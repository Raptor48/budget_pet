## 🚀 Предложения по улучшению

### Приоритет 1: Критичные улучшения UX

#### 1. Быстрый ввод расходов
**Проблема:** Добавление расхода требует открытия диалога, выбора категории, ввода суммы.

**Решение:**
- Добавить плавающую кнопку "Quick Add" на всех страницах
- Горячие клавиши (например, `Ctrl+K` или `/`) для быстрого ввода
- Автодополнение категорий при вводе
- Запоминание последних использованных категорий (MRU - Most Recently Used)
- Шаблоны расходов (например, "Кофе $5", "Обед $15")

**Пример реализации:**
```typescript
// Компонент QuickAddExpense
- Плавающая кнопка в правом нижнем углу
- Модальное окно с поиском категории
- Автодополнение при вводе
- История последних 5 категорий
```

#### 2. Улучшение Dashboard
**Проблема:** Dashboard показывает базовую информацию, но не дает actionable insights.

**Решение:**
- Добавить виджет "Скоро превысите лимит" (категории >80%)
- Прогноз остатка на конец месяца
- Сравнение с предыдущим месяцем (рост/падение расходов)
- Рекомендации ("Вы тратите на 20% больше на Еду, чем в прошлом месяце")
- Индикатор прогресса дня (сколько можно потратить сегодня, чтобы уложиться в бюджет)

#### 3. Уведомления в веб-интерфейсе
**Проблема:** Уведомления о порогах есть только в Telegram боте.

**Решение:**
- Push-уведомления в браузере (Web Push API)
- Toast-уведомления при превышении порогов
- Баннеры на Dashboard для важных алертов
- Настройка порогов уведомлений (не только 50% и 90%)

**Пример:**
```typescript
// Компонент BudgetAlerts
- Toast при добавлении расхода, если превышен порог
- Баннер на Dashboard: "⚠️ Категория 'Еда' превысила 90% лимита"
- Push-уведомление (если разрешено пользователем)
```

### Приоритет 2: Новый функционал

#### 4. Рекуррентные расходы (Recurring Expenses)
**Проблема:** Нужно каждый раз вручную добавлять регулярные расходы (подписки, аренда, коммунальные).

**Решение:**
- Создание шаблонов рекуррентных расходов
- Автоматическое добавление в начале месяца/недели
- Уведомления о предстоящих платежах
- Группировка рекуррентных расходов в отдельную секцию

**Пример:**
```
- Netflix: $15.99/месяц (автоматически 1-го числа)
- Аренда: $1200/месяц (автоматически 5-го числа)
- Интернет: $50/месяц (автоматически 10-го числа)
```

#### 5. Цели и планы (Goals & Savings)
**Проблема:** Нет возможности ставить финансовые цели и отслеживать прогресс.

**Решение:**
- Создание целей (например, "Накопить $5000 к декабрю")
- Отслеживание прогресса
- Автоматический расчет сколько нужно откладывать в месяц
- Визуализация прогресса (progress bar, график)

**Пример:**
```
Цель: Отпуск $3000
Текущий прогресс: $1200 (40%)
Осталось накопить: $1800
Нужно откладывать: $300/месяц (6 месяцев)
```

#### 6. Прогнозы и планирование
**Проблема:** Нет возможности планировать будущие расходы.

**Решение:**
- Календарь запланированных расходов
- Прогноз остатка на конец месяца с учетом запланированных расходов
- "Что если" калькулятор (сколько останется, если потратить X)
- Планирование крупных покупок

#### 7. Сравнение и тренды
**Проблема:** Отчеты показывают только один месяц, нет сравнения.

**Решение:**
- График трендов за 6/12 месяцев
- Сравнение категорий между месяцами
- Выявление паттернов ("Вы всегда тратите больше на Еду в выходные")
- Сезонный анализ (зима vs лето)

**Пример:**
```
График: Расходы на "Еда" за последние 12 месяцев
Тренд: ↑ +15% по сравнению с прошлым годом
Пик: Декабрь ($850) - праздники
```

#### 8. Экспорт и импорт данных
**Проблема:** Нет возможности экспортировать данные для анализа в Excel или импортировать из банков.

**Решение:**
- Экспорт в CSV/Excel
- Экспорт в PDF (красивые отчеты)
- Импорт из CSV (для массового добавления)
- Импорт из банковских выписок (автоматическое распознавание категорий)

### Приоритет 3: Улучшение аналитики

#### 9. Расширенная аналитика расходов
**Проблема:** Базовая аналитика есть, но можно добавить больше insights.

**Решение:**
- Топ категорий по расходам
- Средний чек по категориям
- Частота покупок
- Дни недели с наибольшими расходами
- Время суток (утро/день/вечер) с наибольшими расходами

#### 10. Категоризация и теги
**Проблема:** Одна категория может быть слишком общей.

**Решение:**
- Подкатегории (например, "Еда" → "Рестораны", "Продукты", "Доставка")
- Теги для более гибкой категоризации
- Автоматическая категоризация по ключевым словам в описании
- Правила категоризации (если сумма >$100 и категория "Еда" → "Рестораны")

#### 11. Мультивалютность
**Проблема:** Только доллары США.

**Решение:**
- Поддержка нескольких валют
- Автоматическая конвертация по курсу
- Отдельные бюджеты в разных валютах
- Исторические курсы валют

### Приоритет 4: Мобильная версия и PWA

#### 12. Progressive Web App (PWA)
**Проблема:** Нет мобильного приложения, веб-версия не оптимизирована для мобильных.

**Решение:**
- Превратить Next.js приложение в PWA
- Оффлайн режим (кэширование данных)
- Установка на домашний экран
- Push-уведомления на мобильных устройствах
- Оптимизация для touch-интерфейса

#### 13. Мобильная оптимизация
**Проблема:** Интерфейс может быть неудобен на маленьких экранах.

**Решение:**
- Адаптивный дизайн для всех размеров экранов
- Swipe-жесты для быстрых действий
- Bottom navigation для мобильных
- Упрощенный интерфейс для быстрого ввода

### Приоритет 5: Интеграции и автоматизация

#### 14. Интеграция с банками
**Проблема:** Ручной ввод всех расходов.

**Решение:**
- Интеграция с Plaid/Yodlee для автоматического импорта транзакций
- Поддержка популярных банков
- Автоматическая категоризация транзакций
- Синхронизация балансов счетов

#### 15. Email уведомления
**Проблема:** Уведомления только в Telegram.

**Решение:**
- Еженедельные/месячные отчеты на email
- Уведомления о превышении порогов
- Напоминания о предстоящих платежах
- Настройка частоты уведомлений

#### 16. Календарь платежей
**Проблема:** Нет визуализации когда какие платежи должны быть сделаны.

**Решение:**
- Календарный вид с запланированными платежами
- Цветовая индикация (просрочено, скоро, запланировано)
- Уведомления за N дней до платежа
- Автоматическое создание расходов при наступлении даты
